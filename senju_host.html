📋 SETUP GUIDE
How to connect your server and run bots
HOW IT WORKS
The SENJU HOST panel runs on Netlify (frontend only). To actually execute your Telegram bots, you need to run runner.py on any Linux server or VPS. The runner connects to Firebase in real-time and starts/stops your Python bots based on your commands in this panel. Your bots run FOREVER — no time limit.
1
ENABLE FIREBASE AUTHENTICATION
Go to Firebase Console → Authentication → Sign-in method
Enable Email/Password provider.
Then go to Firestore Database → Create database (start in test mode).
Then go to Storage → Get started.
// Firestore Security Rules — paste these in Firebase Console → Firestore → Rules rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId} { allow read, write: if request.auth != null && (request.auth.uid == userId || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'); } match /bots/{botId} { allow read, write: if request.auth != null && (resource == null || resource.data.ownerUid == request.auth.uid || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'); match /logs/{logId} { allow read: if request.auth != null; allow write: if true; } } match /runners/{runnerId} { allow read: if request.auth != null; allow write: if true; } match /globalEnv/{docId} { allow read: if request.auth != null; allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } } }
2
REGISTER YOUR ADMIN ACCOUNT
Go to the Register tab on the login page and create your account.
The first user to register automatically gets ADMIN role.
Admin has no bot limit and can see all users' bots.
Regular users can only see their own bots (privacy protected).
3
GET FIREBASE SERVICE ACCOUNT KEY
Go to Firebase Console → Project Settings → Service Accounts
Click "Generate new private key" → download the JSON file.
Rename it to service_account.json.
Keep this file PRIVATE — never share or commit to git.
4
SET UP YOUR SERVER / VPS
Any Linux server works (Ubuntu/Debian recommended). Even a cheap $5/mo VPS is fine.
You can also run it on your PC or use Google Cloud Run, Railway, Render, etc.
# Install Python 3 and dependencies sudo apt update && sudo apt install python3 python3-pip -y # Install required Python packages pip3 install firebase-admin requests # Optional — install specific Python versions for your bots sudo apt install python3.12 python3.13 -y
5
DOWNLOAD AND CONFIGURE runner.py
Download runner.py using the button above. Place it in the same folder as service_account.json.
# Your folder structure: ~/senju_runner/ ├── runner.py ← download from this panel └── service_account.json ← from Firebase Console
6
RUN THE RUNNER (KEEP IT RUNNING 24/7)
Start the runner — it will connect to Firebase and wait for your commands from this panel.
# Basic run (stops when you close terminal) cd ~/senju_runner python3 runner.py # Run in background with nohup (stays running after disconnect) nohup python3 runner.py > runner.log 2>&1 & echo "Runner PID: $!" # OR use screen (recommended — lets you attach/detach) screen -S senju python3 runner.py # Press Ctrl+A then D to detach # screen -r senju to reattach # OR use systemd service for auto-start on reboot: # sudo nano /etc/systemd/system/senju-runner.service
7
DEPLOY YOUR BOT
1. Go to Deploy page in this panel
2. Enter your bot name and Telegram bot token (from @BotFather)
3. Upload your .py bot file
4. Click Deploy & Start Bot
5. The runner.py on your server will download and execute the bot automatically!
6. Watch the Live Logs page for real-time output.
8
DEPLOY TO NETLIFY (PANEL HOSTING)
The panel HTML file runs completely on Netlify for free. No server needed for the panel itself.
# Method 1: Drag and drop 1. Go to netlify.com → Log in 2. Drag this HTML file to the deploy box 3. Done! Your panel is live. # Method 2: GitHub 1. Push this HTML file as index.html to GitHub repo 2. Connect repo to Netlify 3. Auto-deploys on every push
⚡ PRO TIPS
• Bots run with NO TIME LIMIT — as long as runner.py is running on your server
• If a bot crashes, runner.py auto-restarts it based on your restart policy
• All bot output (print, errors) is streamed to Live Logs in real-time
• Environment variables you set in the panel are auto-injected into your bot
• BOT_TOKEN env var is always injected automatically so your bot can read it
• Multiple bots can run at the same time — runner handles them all
