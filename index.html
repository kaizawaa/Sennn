HOW SENJU HOST WORKS
This panel (hosted on Netlify) + Firebase stores your bot configs & files.
A worker.py script on your VPS reads Firebase and actually runs your bots. No time limit.
1
Get a VPS or Server
Your bot runs on a VPS (Virtual Private Server). Free options include:
• Oracle Cloud Free Tier — Always Free VPS (recommended)
• Google Cloud Free Tier — e1-micro instance
• DigitalOcean, Vultr, Contabo (paid, from $3/mo)
• Your own PC (Windows/Linux/Mac) — runs while PC is on
💡 Oracle Cloud gives you a free forever VPS — perfect for 24/7 bot hosting with no time limit!
2
Set Up Python on Your VPS
SSH into your VPS and install Python + required packages:
sudo apt update && sudo apt install python3 python3-pip -y pip3 install firebase-admin python-telegram-bot requests
3
Get Firebase Service Account Key
1. Go to console.firebase.google.com
2. Open your project → Project Settings
3. Click Service Accounts tab
4. Click Generate new private key
5. Save the downloaded file as serviceAccountKey.json
⚠ Never share your serviceAccountKey.json! Keep it private on your VPS only.
4
Download & Run the Worker Script
Download worker.py from the Worker Script page and upload it to your VPS alongside serviceAccountKey.json:
scp worker.py serviceAccountKey.json user@your-vps-ip:~/senju_host/ ssh user@your-vps-ip cd ~/senju_host python3 worker.py
💡 To keep it running forever, use: nohup python3 worker.py &
Or use screen/tmux: screen -S senju python3 worker.py
5
Upload Your Bot & Deploy
1. Go to Deploy Bot in the sidebar
2. Enter your bot name and token
3. Upload your .py bot file OR paste the code
4. Click Deploy to Firebase Now
5. Go to My Bots and click ▶ START
6. Your worker on the VPS will detect the command and start your bot!
✅ Once deployed, your bot runs 24/7 with no time limit. The worker auto-restarts it on crashes!
6
Monitor & Manage
• Logs page — see real-time logs from your bot (sent from worker to Firebase)
• Monitor page — see resource usage of all running bots
• Start/Stop/Restart — instantly control your bot from anywhere
• Environment — update env variables (bot will reload on restart)
• All data synced to Firebase — access from any device, anywhere
7
Firebase Storage Setup (for file uploads)
To enable bot file uploads via the panel, enable Firebase Storage:
1. Firebase Console → Storage → Get Started 2. Choose a region → Done 3. Go to Storage Rules and set: allow read, write: if true; (for testing) 4. For CORS, run in Google Cloud Shell: gsutil cors set cors.json gs://senju-vip-hosting.appspot.com
💡 If Storage isn't set up, use the "Paste Code" tab to deploy your bot — it stores the code directly in Firestore!
⚡ WORKER SCRIPT
Download and run on your VPS to actually execute your bots
WHAT THE WORKER DOES
✅ Connects to Firebase Firestore
✅ Watches for START/STOP commands
✅ Downloads your bot .py file
✅ Runs bot as Python subprocess
✅ Sends logs back to Firebase
✅ Auto-restarts on crash
✅ Handles multiple bots at once
✅ No time limit — runs forever
✅ Reports real-time status
REQUIREMENTS
• Python 3.10+
• firebase-admin library
• serviceAccountKey.json
• VPS / Server / PC running 24/7
• Internet connection
pip3 install firebase-admin
